Creative Process

I used two buttons, a speaker, and an LCD screen for my project. The buttons record the user's input while playing, with one button being a continue and the other a stop turn. The button inputs tell the LCD to update and the speaker to play a sound. Both the speaker and the screen provide feedback to the user, with the screen displaying the score and the speaker playing noises depending on whether the user is losing, continuing, or getting a high score.

I had never worked with Arduino and breadboards. One of the first challenges I faced was getting the screen to work correctly; when wiring it up, I first forgot to add a resistor to one of the rows, making the screen malfunction. Another problem I came across was that the code for the game wasn't working as intended with my inputs. Although it seemed like a coding issue, it was actually a wiring issue. I overcame these challenges by ensuring my wiring was correct before testing.

My code uses the LiquidCrystal library, which has functions like lcd.clear(), lcd.print(), and lcd.setCursor(). Each is self-explanatory: clear() clears the screen, print() displays text or numbers, and setCursor() positions the text. Arduino has the convenient loop() function that does precisely what its name suggestsâ€”it loops continuously, allowing my program to change and respond dynamically. In my loop function, I have moveChicken() and stopChicken(), and each function's meaning closely follows its name. The moveChicken() function is conditional on continuing the game, where a crash value is checked with a random number for each move.
